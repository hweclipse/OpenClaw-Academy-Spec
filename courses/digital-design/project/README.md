# Mini SoC — 课程长期项目

## 项目概述

Mini SoC 是本课程的长线项目，贯穿全部 10 章。每一章为 Mini SoC 贡献一个模块，最终集成为一个可运行的系统。

## 目标架构

```
┌─────────────────────────────────────────────┐
│                  Mini SoC                     │
│                                               │
│  ┌─────────┐   ┌──────────┐   ┌───────────┐ │
│  │ Control  │──→│ Datapath │──→│ Register  │ │
│  │  Unit    │   │  (ALU)   │   │   File    │ │
│  └────┬─────┘   └──────────┘   └───────────┘ │
│       │                                       │
│  ┌────┴─────┐   ┌──────────┐   ┌───────────┐ │
│  │ Program  │   │   Data   │   │    I/O    │ │
│  │  Memory  │   │  Memory  │   │ Controller│ │
│  └──────────┘   └──────────┘   └───────────┘ │
│                                               │
└─────────────────────────────────────────────┘
```

## 里程碑

| 编号 | 里程碑 | 对应章节 | 交付物 |
|------|--------|----------|--------|
| M1 | 基本逻辑单元库 | Ch02 | `gates.v` — 参数化门级元件库 |
| M2 | 寄存器与计数器 | Ch05 | `regfile.v`, `counter.v` — 参数化寄存器/计数器 |
| M3 | ALU | Ch07 | `alu.v` — 8 位 ALU（8 种运算） |
| M4 | 外设控制器 | Ch09 | `uart_tx.v` — 基于 FSM 的简单 UART 发送器 |
| M5 | Mini SoC 集成 | Ch10 | `mini_soc.v` — ALU + CU + REGFILE + MEM 顶层 |

## 代码规范

- 所有模块遵循 `COURSE_SPEC.md` 中的注释规范
- 模块文件头必须标注对应的章节和里程碑
- 接口信号优先使用参数化位宽

## 测试策略

- 每个模块配有独立 Testbench（`tb_<module>.v`）
- 里程碑通过集成测试验收（Testbench 必须包含边界条件）
- M5 集成测试运行一个简单程序（如斐波那契数列计算）
