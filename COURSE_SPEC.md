# COURSE_SPEC.md

OpenClaw Academy 课程内容结构规范 v1.0

---

## 1. 设计原则

- **统一格式**：所有课程的所有章节使用相同的文档格式
- **逐步深入**：知识 → 练习 → 实验 → 总结，不可跳跃
- **可验证**：每个实验有明确的验收标准
- **可追溯**：每个知识点可追溯到对应的实验和项目

---

## 2. 章节文档规范

每个章节必须包含以下四个核心文档：

### 2.1 knowledge.md — 知识点讲解

```markdown
# <章节编号> <章节标题> — 知识点

## 学习目标
- 列出本章应掌握的知识点（3-7 条）
- 使用可验证的动词：能够解释、能够设计、能够分析

## 前置知识
- 列出依赖的前序章节或知识点

## 核心概念

### <概念 1>
- 解释
- 图示（如有，放入 assets/）
- 关键公式或规则

### <概念 2>
同上

## 本章术语表
| 中文 | English | 说明 |
|------|---------|------|
| 组合逻辑 | Combinational Logic | 输出仅依赖当前输入 |

## 延伸阅读
- 参考书目
- 参考链接
```

### 2.2 practice.md — 练习题

```markdown
# <章节编号> <章节标题> — 练习

## 练习 1：<题目>
### 题目
<问题描述>

### 提示
<可选提示>

### 答案
<参考答案，由 Codex 填写>

---

## 练习 N
（同上）

## 自测清单
- [ ] 我能 <验证目标 1>
- [ ] 我能 <验证目标 2>
```

### 2.3 experiment.md — 实验指导

```markdown
# <章节编号> <章节标题> — 实验

## 实验目标
- 一句话说明本实验要完成什么

## 前置条件
- 已完成的章节
- 需要的工具链（iverilog / Vivado / Quartus 等）

## 背景
- 实验背景知识（与 knowledge.md 互补但不重复）

## 任务

### 任务 1：<任务名称>
- 输入规格
- 输出规格
- 行为描述
- 约束条件

### 任务 2
（同上）

## 验收标准
- [ ] 任务 1：Testbench 通过
- [ ] 任务 2：波形正确
- [ ] 代码风格符合规范
- [ ] 已在长期项目中集成

## 实验产出
- `code/<module>.v` — RTL 实现
- `code/tb_<module>.v` — Testbench
- `code/<module>_wave.vcd` — 波形截图或 VCD

## 常见问题
- Q: ...
  A: ...
```

### 2.4 review.md — 章节总结

```markdown
# <章节编号> <章节标题> — 总结

## 知识点回顾
- 列出本章核心知识点

## 实验总结
- 完成实验数 / 总实验数
- 遇到的问题与解决方案
- Debug 耗时

## 项目集成
- 本章成果如何集成到长期项目
- 项目的哪个里程碑受益

## 遗留问题
- 尚未完全理解的内容
- 需要后续章节补充的知识

## 下一章预告
- 下一章内容概述
- 需要提前准备的工具或知识
```

## 3. 代码规范

### 3.1 代码块标记

所有代码块必须标注语言：

```markdown
​```verilog
// Verilog 代码
​```

​```systemverilog
// SystemVerilog 代码
​```

​```bash
# Shell 命令
​```

​```tcl
# Tcl / Vivado 脚本
​```
```

### 3.2 Verilog 模块注释规范

```verilog
// ============================================
// Module    : <module_name>
// Author    : User
// Chapter   : <chapter_slug>
// Date      : <YYYY-MM-DD>
// Project   : <project_name>
// ============================================
// Description:
//   <一句话描述模块功能>
//
// Interface:
//   clk    - 时钟信号
//   rst_n  - 异步复位（低有效）
//   ...
//
// Parameters:
//   WIDTH  - 数据位宽（默认 8）
// ============================================

module <module_name> #(
    parameter WIDTH = 8
)(
    input  wire             clk,
    input  wire             rst_n,
    // ...
    output wire [WIDTH-1:0] result
);
    // TODO: User implementation
endmodule
```

### 3.3 文件头注释

每个代码文件必需文件头：

```verilog
// ============================================
// OpenClaw Academy
// Course     : <course_name>
// Chapter    : <chapter_slug>
// ============================================
```

## 4. 用户代码标记

### 4.1 代码区域划分

代码模板使用注释标记用户编辑区域：

```verilog
// === BEGIN USER CODE ===
// 在此处实现你的设计
// === END USER CODE ===
```

### 4.2 完成标记

User 提交完成的代码后，Codex 不得修改。识别方式：
- User 在 commit message 中标注 `[COMPLETE] <chapter_slug>`
- Tracker 中章节状态变为 `completed`

---

## 5. 实验验收标准规范

每个实验必须满足以下通用标准：

| 标准 | 说明 |
|------|------|
| 编译通过 | 无 Error，Warnings 数量可控 |
| Testbench 通过 | 所有 test case PASS |
| 波形正确 | 关键信号时序符合预期 |
| 代码风格 | 缩进一致、命名规范、有注释 |
| 项目集成 | 模块在顶层项目中实例化并通过集成测试 |

---

## 6. 扩展规则

- 新增文档类型时，先在本文档中注册格式约定
- 不得在章节中插入非标准格式的文档
- 跨课程共享内容放入 `shared/`，在课程文档中用相对路径引用
